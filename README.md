# dev-of-info-sec-tools
##Encryptor
1. Постановка задачи

Разработать программу для защиты конфиденциальных данных пользователя путем шифрования всех файлов в указанной директории и всех её поддиректориях. Программа должна работать в консольном режиме и принимать на вход путь к папке и пароль. Для доступа к исходным данным требуется их дешифрование тем же паролем. Ключевые требования:

Язык реализации: C++.
Интерфейс: консоль.
Шифрование: AES-256 (например, с помощью библиотеки Crypto++).
Обход файлов: рекурсивный (с использованием Qt).
Архитектура: основной класс для шифрования/дешифрования должен быть реализован как Singleton.
Надежность: отсутствие утечек памяти, обработка исключений.
2. Предлагаемое решение

Идея решения

Программа получает от пользователя путь к корневой папке и пароль. На основе пароля с помощью функции формирования ключа (KDF - Key Derivation Function, например, PBKDF2) генерируется надежный ключ шифрования. Затем программа рекурсивно обходит все файлы в заданной папке и для каждого файла выполняет операцию (шифрование или дешифрование) с использованием алгоритма AES-256. Метаданные (например, вектор инициализации) сохраняются вместе с зашифрованными данными или в имени файла, чтобы обеспечить возможность дешифрования.

Архитектура ПО

Программа будет состоять из двух основных классов, что обеспечивает четкое разделение ответственности (Single Responsibility Principle):

CryptoManager (Синглтон): Отвечает за криптографические операции.
Приватный конструктор и статический метод instance() для реализации паттерна Singleton.
Методы encryptFile и decryptFile, принимающие пути к файлам и ключ.
Метод deriveKeyFromPassword для генерации ключа AES-256 из пароля (с использованием соли и PBKDF2).
DirectoryProcessor: Отвечает за навигацию по файловой системе.
Метод processDirectory, который рекурсивно обходит все подпапки.
Для каждого найденного файла вызывает соответствующий метод CryptoManager.
UML диаграмма классов:

classDiagram
    class CryptoManager {
        -static CryptoManager* instance_
        -CryptoManager()
        +static CryptoManager* instance()
        +vector~byte~ deriveKeyFromPassword(string password, vector~byte~ salt)
        +void encryptFile(string filePath, vector~byte~ key)
        +void decryptFile(string filePath, vector~byte~ key)
    }
    class DirectoryProcessor {
        +void processDirectory(string rootPath, vector~byte~ key, Operation op)
    }
    class Program {
        +main()
    }
    Program --> DirectoryProcessor : uses
    Program --> CryptoManager : uses
    DirectoryProcessor --> CryptoManager : calls encrypt/decrypt
    
Паттерн Singleton в CryptoManager гарантирует, что ресурсы криптографической библиотеки (если они будут) инициализируются только один раз, и предоставляет единую точку доступа к функциям шифрования.
Разделение классов позволяет легко тестировать криптографическую логику независимо от файловой и наоборот.
Используемые технологии
Библиотека Crypto++:
Суть: Мощная и зрелая библиотека на C++ для криптографии. Предощает реализации множества алгоритмов, включая AES и PBKDF2.
Задача: Мы используем Crypto++ для низкоуровневых криптографических операций, чтобы не реализовывать их вручную, что было бы небезопасно. Она обеспечит корректную и безопасную работу AES-256.
Шаги использования:
Установка библиотеки (например, через пакетный менеджер vcpkg, Conan или сборка из исходников).
Подключение необходимых заголовочных файлов (#include <aes.h>, <modes.h>, <filters.h>, <pbkdf2.h> и др.).
Линковка с библиотекой (libcryptopp.a или cryptopp.lib).
Библиотека Qt (частично: QtCore):
Суть: Qt — фреймворк для разработки кроссплатформенных приложений. Модуль QtCore предоставляет классы для работы с файловой системой, не зависящие от ОС.
Задача: Используем QDir и QFileInfo для рекурсивного обхода директорий. Это кроссплатформенно и удобнее, чем стандартные функции C++17 (std::filesystem), особенно если в дальнейшем проект будет расширяться.
Шаги использования:
Установка Qt (или только модуля QtCore).
Настройка проекта (например, в CMakeLists.txt) для нахождения Qt и линковки с Qt::Core.
Использование QDirIterator с флагом QDirIterator::Subdirectories для рекурсивного обхода.
3. Инструкция пользователя

Приложение является консольным и запускается из терминала (командной строки).

Формат вызова:

```bash
./folder_cipher <command> <folder_path> <password>
<command>: encrypt или decrypt.
<folder_path>: Путь к папке, которую нужно обработать.
<password>: Пароль для шифрования/дешифрования.
```
Примеры использования:

Шифрование папки ./my_secrets:
```bash
./folder_cipher encrypt ./my_secrets my_strong_password
```
Программа рекурсивно обойдет все подпапки в my_secrets и зашифрует каждый найденный файл. Исходные файлы будут заменены на зашифрованные (например, с расширением .enc). В случае ошибки (неверный пароль, нет прав доступа) программа выведет сообщение и завершит работу.
Дешифрование папки ./my_secrets:
```bash
./folder_cipher decrypt ./my_secrets my_strong_password
```
Программа найдет все файлы с расширением .enc (или зашифрованные файлы без расширения, если мы сохраним метаданные иначе) и расшифрует их, восстановив исходное содержимое и имя.
4. Тестирование

План тестирования

Тестирование проводится для проверки функциональности, надежности и обработки ошибок.

ID теста	Действие (Входные данные)	Ожидаемая реакция программы
TC-01	Запуск с некорректным количеством аргументов (./folder_cipher)	Вывод справки с информацией об использовании и завершение работы с кодом ошибки.
TC-02	Шифрование существующей папки с одним текстовым файлом.	Программа успешно завершается. Содержимое файла становится нечитаемым (бинарным).
TC-03	Дешифрование папки из TC-02 с тем же паролем.	Программа успешно завершается. Содержимое файла восстанавливается до исходного текста.
TC-04	Дешифрование папки с зашифрованными данными с неправильным паролем.	Программа выбрасывает исключение (например, std::runtime_error об ошибке аутентификации), ловит его, выводит сообщение об ошибке и завершается, не повреждая файлы.
TC-05	Шифрование папки с глубокой вложенностью подпапок и разными типами файлов (.txt, .jpg, .bin).	Все файлы во всех подпапках успешно шифруются.
TC-06	Шифрование пустой папки.	Программа успешно завершается, ничего не делая (или выводит сообщение, что файлы не найдены).
TC-07	Запуск для несуществующего пути к папке.	Программа выводит сообщение об ошибке "Указанная папка не существует" и завершается.
TC-08	Шифрование/дешифрование файла, к которому нет прав на чтение/запись.	Программа обрабатывает исключение от Qt (QFile::NoPermission), выводит предупреждение и продолжает работу с остальными файлами.
TC-09	Проверка утечек памяти (запуск под Valgrind или аналогичным инструментом).	Отсутствие потерь памяти при выполнении любой из операций.
